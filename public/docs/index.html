Introduction

This is a sample Travel API. Navigate to the endpoints to see the responses.

Base URL: http://localhost:8000/api/v1

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

To authenticate requests, include an Authorization header with the value "Bearer 1|uRQnDZWtlDj1rzqtRrADcaFEInTGWvFBAbejXTAW".

All authenticated endpoints are marked with a requires authentication badge in the documentation below.

You can retrieve your token by visiting your dashboard and clicking Generate API token.

Admin Controls

Create a new Travel.

Example request:
curl --request POST \
    "http://localhost:8000/api/v1/api/v1/admin/travels" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"is_public\": false,
    \"name\": \"id\",
    \"description\": \"non\",
    \"number_of_days\": 8
}"

const url = new URL(
    "http://localhost:8000/api/v1/api/v1/admin/travels"

    "description": "non",
    "number_of_days": 8

Request      

POST api/v1/admin/travels/{travel_id}/tours

Headers

Content-Type      

Example: application/json

Accept      

Example: application/json

URL Parameters

travel_id   string   

The ID of the travel. Example: 996b10c9-31d8-492f-9c57-fd2a57fd1c8d

Body Parameters

name   string   

Must not be greater than 100 characters. Example: hpymwb

starting_date   string   

Must be a valid date. Example: 2023-06-19T22:21:28

ending_date   string   

Must be a valid date. Must be a date after starting_date. Example: 2045-05-02

price_in_cents   number   

Must be at least 0. Example: 64
